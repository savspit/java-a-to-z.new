Shared Object Collision
Eсли два и более потока имеют доступ к одному объекту без использования ключевого слова volatile или синхронизации,
то обновление этого объекта из данных потоков будет невидимо из других потоков.

Решение проблемы: использование ключевого слова volatile

Race Condition Collision
Если два и более потока имеют доступ к одному объекту и более, чем один поток обновляет переменные данного объекта без должной синхронизации,
то это обновление будет иметь некорректный результат

Решение проблемы: использование ключевого слова synchronized (синхронизированных блоков)

Описание классов:
- TestObject - объект для тестирования
- SharedObjectCollision - метод main() выполняет и выводит на консоль результат коллизии
- RaceConditionCollision - метод main() выполняет и выводит на консоль результат коллизии
